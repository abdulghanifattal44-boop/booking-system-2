# Сценарии использования (Use Cases)

## Акторы
- **Guest** (неавторизованный)
- **User** (клиент)
- **Admin** (администратор)

## UC-01 Просмотр доступности (Guest)
1. Открыть Customer Portal
2. Выбрать Organization → Branch → Resource
3. Выбрать дату
4. Получить список доступных слотов

## UC-02 Регистрация (Guest → User)
- Создание аккаунта → получение JWT  
- Email уже используется → 409/400

## UC-03 Вход (Guest → User/Admin)
- Ввод email/пароль → JWT  
- Ошибка → 401

## UC-04 Создание бронирования (User)
- Выбор слота → Confirm Booking  
- Если не вошёл: UI просит Login/Register на подтверждении  
- Слот занят → 409 → сообщение «Слот уже забронирован»

## UC-05 Мои бронирования (User)
- Список бронирований пользователя (ресурс/время/статус)

## UC-06 Отмена бронирования (User)
- Проверка правил → CANCELLED  
- Нельзя → 403

## UC-A01 Вход админа (Admin)
- `/admin/login` → Dashboard

## UC-A02..A06 (Admin)
- CRUD организаций/филиалов/ресурсов
- Генерация слотов
- Управление бронированиями и статусами
